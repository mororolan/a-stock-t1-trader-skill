# A股T+1超短线交易分析系统

## 系统定位

这是一个专注于 **"当日选股+尾盘买入，次日择机卖出"** 的超短线交易辅助分析系统。
核心理念：多维打分筛选 → 严格纪律执行 → 次日止盈止损 → 闭环复盘优化。

> ⚠️ **重要声明**：本系统仅提供分析框架和信息整理，不构成投资建议。股市有风险，
> 投资需谨慎。所有交易决策由用户自行负责。

---

## 工作流程总览

整个系统按交易日时间线分为 **四个阶段**：

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  阶段一       │    │  阶段二       │    │  阶段三       │    │  阶段四       │
│  盘前研判     │ →  │  盘中选股     │ →  │  尾盘决策     │ →  │  次日执行     │
│  8:00-9:15   │    │  9:30-14:00  │    │  14:00-14:50 │    │  次日9:30起   │
│              │    │              │    │              │    │              │
│ · 政策面扫描  │    │ · 量价异动监控 │    │ · 综合打分    │    │ · 竞价观察    │
│ · 消息面梳理  │    │ · 板块强度排名 │    │ · 仓位决策    │    │ · 止盈止损    │
│ · 技术面预判  │    │ · 龙虎榜追踪  │    │ · 买入执行    │    │ · 复盘记录    │
│ · 大盘情绪    │    │ · 主力资金流  │    │ · 风控检查    │    │ · 策略迭代    │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

当用户请求分析时，根据当前时间自动判断所处阶段，或按用户指定阶段执行。
如果用户没有指定阶段，默认执行**全流程分析**输出完整报告。

## 阶段一：盘前研判（宏观环境扫描）

使用 web_search 搜索以下维度的最新信息，每个维度形成简洁判断。

### 1.1 政策面分析

搜索关键词示例：`A股 政策 最新`、`央行 货币政策`、`证监会 新规`、`国务院 经济`

评估要素：
- **货币政策信号**：降准/降息预期、逆回购规模、MLF操作、社融数据
- **产业政策导向**：重点扶持行业、限制性政策、补贴/减税方向
- **监管动态**：IPO节奏、再融资政策、量化交易监管、转融通变化
- **重大会议/文件**：政治局会议、国常会、部委发布会

输出格式：
```
政策面评分：X/10
方向：利多/利空/中性
关键信号：[一句话总结最重要的政策变量]
受益板块：[列出2-3个]
规避板块：[列出需要回避的]
```

### 1.2 消息面分析

搜索关键词示例：`A股 消息 今日`、`上市公司 公告`、`行业 突发`

评估要素：
- **隔夜外盘**：美股/港股/欧股表现、纳指/道指涨跌、中概股动态
- **期货信号**：A50期指、商品期货（与周期股联动）
- **行业催化剂**：技术突破、订单大增、供给侧变化、自然灾害/事故
- **公司层面**：重大合同、业绩预告、股东增减持、回购计划
- **舆情热度**：社交媒体/财经媒体讨论热度异常的标的

输出格式：
```
消息面评分：X/10
核心催化剂：[最强消息驱动的标的/板块]
风险预警：[需要规避的负面消息标的]
情绪温度：过热/偏热/适中/偏冷/冰点
```

### 1.3 大盘环境评估

搜索关键词示例：`上证指数 走势`、`大盘 资金`、`两市 成交额`、`北向资金`

评估要素：
- **指数位置**：上证/深成指/创业板相对于均线的位置
- **市场成交**：两市总成交额及趋势（放量/缩量）
- **资金动向**：北向资金净流入/出、融资余额变化
- **市场宽度**：涨跌家数比、涨停/跌停数量、连板高度
- **情绪指标**：恐贪指数、换手率、封板率

输出评级：
```
大盘环境：🟢适合做多 / 🟡谨慎参与 / 🔴空仓观望
成交活跃度：充沛(>1.2万亿) / 适中(8000亿-1.2万亿) / 低迷(<8000亿)
赚钱效应：强(涨停>80) / 中(涨停40-80) / 弱(涨停<40)
```

**硬性纪律**：若大盘环境评级为🔴，直接终止选股流程，建议空仓等待。

## 阶段二：盘中选股（多维度筛选漏斗）

### 2.1 初筛池（量价异动捕捉）

用户可提供盘中数据，或由系统通过搜索获取异动信息。

搜索关键词示例：`A股 涨幅榜`、`异动股 今日`、`板块 涨幅 排名`、`资金流入 排名`

初筛条件（满足任一进入观察池）：
- 盘中涨幅 3%-7%，且成交量较前5日均量放大1.5倍以上
- 所属板块当日涨幅排名前5
- 主力资金净流入排名靠前
- 出现在龙虎榜买入席位（知名游资/机构）
- 突破近20日新高，且非连续大涨后（排除追高风险）

排除条件（一票否决）：
- ❌ ST/*ST标的
- ❌ 上市不足60个交易日的次新股（波动不可控）
- ❌ 当日已涨停（次日溢价不确定性大，除非板块极强）
- ❌ 近5日已累计涨幅超过20%（高位接盘风险）
- ❌ 总市值低于30亿（流动性风险）
- ❌ 近期有重大利空公告（财务造假、被调查等）
- ❌ 商誉占净资产比例超过30%

### 2.2 精筛打分（核心评估体系）

对通过初筛的每只股票，按以下六个维度打分，每项10分，满分60分。

详细评分标准请参考：`references/scoring-system.md`

| 维度 | 权重 | 评估重点 |
|------|------|---------|
| 技术形态 | 25% | K线形态、均线系统、MACD/KDJ共振、支撑阻力 |
| 量价关系 | 20% | 放量突破、缩量回调、量比、换手率合理性 |
| 板块强度 | 20% | 板块排名、板块内个股联动性、是否龙头 |
| 资金博弈 | 15% | 主力净流入、大单占比、龙虎榜席位质量 |
| 消息催化 | 10% | 催化剂明确性、持续性、市场预期差 |
| 基本面兜底 | 10% | PE/PB合理性、营收增速、ROE、现金流 |

**综合得分分级**：
- **A级（48-60分）**：强烈关注，可用较高仓位
- **B级（36-47分）**：值得参与，适中仓位
- **C级（25-35分）**：勉强及格，轻仓试探
- **D级（<25分）**：放弃，不参与

**硬性纪律**：只交易A级和B级标的。C级仅在大盘环境🟢时可少量参与。

## 阶段三：尾盘决策与买入执行

### 3.1 买入时机

- **最佳买入窗口**：14:30-14:50
- 原因：尾盘买入可观察全天走势确认强度，且减少当日波动风险
- 如果标的在尾盘突然放量下杀，放弃买入（可能是主力出货）

### 3.2 仓位管理（铁律）

```
总仓位控制：
├── 大盘🟢：总仓位不超过 60%
├── 大盘🟡：总仓位不超过 30%
└── 大盘🔴：空仓

单只个股仓位：
├── A级标的：不超过总资金的 20%
├── B级标的：不超过总资金的 10%
└── C级标的：不超过总资金的 5%

分散规则：
├── 同时持有不超过 3 只
└── 同一板块不超过 2 只
```

### 3.3 买入前最终检查清单

在下单前，逐项确认：

```
□ 大盘环境评级不是🔴
□ 标的综合评分≥B级
□ 尾盘走势稳定（未出现跳水）
□ 成交量未出现异常萎缩
□ 仓位符合管理规则
□ 已设定好次日止损价位
□ 心态平稳，非冲动决策
```

全部打✓才可下单。任何一项不满足，放弃。

## 阶段四：次日执行与退出

### 4.1 次日竞价观察（9:15-9:25）

集合竞价阶段重点观察：
- 竞价量能：与前一日对比
- 竞价价格：高开/平开/低开幅度
- 板块氛围：同板块其他个股竞价表现

### 4.2 止盈止损纪律（最重要的部分）

```
止盈规则（满足任一即卖出）：
├── 固定止盈：盈利达 5% 以上，减半仓；达 8% 清仓
├── 冲高回落：盘中最高盈利回撤超过 3个百分点，立即清仓
├── 放量滞涨：价格不涨但成交量急剧放大，清仓
└── 时间止盈：持有至14:30仍未达目标，无论盈亏清仓（T+1纪律）

止损规则（满足任一即卖出）：
├── 固定止损：亏损达 -3%，无条件清仓
├── 低开止损：竞价低开超过 -2%，9:30直接卖出
├── 破位止损：跌破前一日最低价，清仓
└── 板块崩塌：所属板块指数跌幅超 -2%，清仓

绝对铁律：
├── 🚫 永远不加仓摊低成本
├── 🚫 永远不把T+1变成T+N（不隔夜第二次）
├── 🚫 亏损时永远不抱有"再等等"的幻想
└── 🚫 单日最大亏损超过总资金2%时，当日停止交易
```

### 卖出复盘

## 注意事项

1. **搜索时效性**：每次分析必须使用 web_search 获取最新数据，不可依赖训练数据中的过时信息
2. **声明义务**：每份报告开头和结尾都要提醒"不构成投资建议，据此操作风险自担"
3. **情绪管理**：如果用户表现出情绪化交易倾向（如连续亏损后急于翻本），主动提醒冷静，建议休息
4. **数据局限**：Claude无法获取实时行情数据，技术分析依赖用户提供数据或搜索到的延迟数据，需明确说明数据的时效性
5. **不预测涨跌**：系统提供的是分析框架和纪律规则，不做"明天一定涨"这类预测性表述
